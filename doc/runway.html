Sage Quiz runway documentation

This document captures the current product direction for Sage Quiz as a SageCode subdomain experience.

Product goals

  • Serve as a modern quiz experience for the SageCode community.
  • Support a subdomain such as quiz.sagecode.org.
  • Offer a clean navigation flow with Home, Login, Register, Quizzes, and Results.
  • Provide a responsive dark design with a hamburger menu on smaller screens.

Current experience

  • Users can register with email, password, Discord username, and profile URL.
  • Users can sign in with email and password.
  • Quizzes are listed with code, short name, and taker count.
  • Public results are shown in reverse chronological order.
  • Authenticated users can review their own recent results.

Technical rollout

  • Frontend: Svelte + Vite.
  • Backend: Cloudflare Worker.
  • Data: Cloudflare D1 database.
  • Static assets: Worker assets binding.

Next steps

  • Point the subdomain DNS record to the Cloudflare Worker.
  • Use the SageCode icon and logo assets from the CSE project in the branding.
  • Expand the quiz catalog with more quiz JSON files over time.
  • Add stronger password policies and email verification later if needed.
