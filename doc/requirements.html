Requirements

This Quiz App is designed to evaluate and validate the knowledge and understanding of Software Engineering and Programming Fundamentals. This app is designed to demonstrate our ability to code but also useful for Sage-Code Laboratory.

Functional Requirements

Registration/Login

  • Users should be able to create a new account.
  • Users should be able to log in to their existing account.
  • Users should be able to change password.
  • Users should have the right to be forgoten.

Quiz Taking

  • Users should be able to select the course they wish to take the quiz for.
  • Quiz questions should jave 1 up to 4 possible answers.
  • Most questions have one or more true options and the rest false.
  • Questions should be randomly generated from a question bank to provide different quiz experiences for each user.
  • Questions should be organized in sections, and order of sections is establish not random, while questions in a sections are random
  • After completing the quiz, users should receive a score and a report detailing which questions they answered correctly or incorrectly.

Quiz results

  • Users should be able to see partial results for each section.
  • Users should opt out of the quiz early if do not pass a section.
  • Users should be able to navigate back and forward in a section.
  • Users should be able to review total points received.
  • Users should be able to receive PASS/FAIL status on-line.
  • Users should be able to optional make the result public.
  • Users should have the right erase results for a quiz.

Non-Functional Requirements

Interface Design

  • The user interface should be user-friendly and responsive.
  • The interface should showcase Sage-Code branding theme.

Security Requirements

  • User passwords should be securely stored using a hashing algorithm.
  • User sessions should be authenticated and encrypted.
  • The quiz system should be protected against SQL injections and other possible security threats.
  • User account should be validated after registration by a mentor. If the account is not validated, the user can't take a quiz.

Performance Requirements

  • The quiz system should be fast and responsive.
  • The system should be able to handle multiple concurrent users.
  • The quiz system should be able to handle a large number of questions.

Conclusion

The Sage-Code Ask App should be designed with both functional and non-functional requirements in mind to provide a secure, user-friendly, and responsive quiz experience for users.
