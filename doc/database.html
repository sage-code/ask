Ask Cloudflare Database

The Cloudflare D1 database stores user accounts and quiz results. Quiz questions are loaded from JSON files in public/quizzes/, not from the database.

Tables

users

CREATE TABLE users (
  id TEXT PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  name TEXT NOT NULL,
  role TEXT NOT NULL,
  created_at TEXT NOT NULL
);

results

CREATE TABLE results (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  quiz_id TEXT NOT NULL,
  quiz_title TEXT NOT NULL,
  score INTEGER NOT NULL,
  total INTEGER NOT NULL,
  answers TEXT NOT NULL,
  taken_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

Data model

  • users: stores mentors and students.
  • results: stores quiz attempts and performance metrics.

Saved result payload

The API saves these fields from the app:

  • user_id - registered user identifier.
  • quiz_id - quiz slug string.
  • quiz_title - display title.
  • score and total - scoring information.
  • answers - JSON-encoded answer indexes.
  • taken_at - UTC timestamp.

Cloudflare D1 setup

  1. Create a D1 database in Cloudflare dashboard.
  2. Add the D1 binding in wrangler.toml:
[[d1_databases]]
binding = "DATABASE"
database_name = "ask_db"

Then run the schema in scripts/schema.sql.

API endpoints

  • POST /api/register - register or return existing user.
  • POST /api/results - save quiz attempt results.
  • GET /api/results?userId=<id> - list recent quiz results for a user.
